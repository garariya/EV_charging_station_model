# Contributing to EV Charging Station Classification

First off, thank you for considering contributing to this project! It's people like you that make predicting EV charging infrastructure better.

## How Can I Contribute?

### Reporting Bugs
If you find a bug in the source code or a mistake in the documentation, you can help us by submitting an issue to our GitHub Repository.

### Enhancing Models
We welcome improvements to our initial models, such as our Random Forest and XGBoost predictors. You can:
- Suggest new feature engineering techniques or variables to include.
- Improve hyperparameter settings.
- Add scripts for more extensive EDA (Exploratory Data Analysis).

### Application UI Improvements
Our Streamlit application can always be made more intuitive. Feel free to contribute by:
- Adding visualization charts for exploratory data analysis.
- Enhancing the responsive design and user flow.
- Reporting or fixing UI layout issues.

## Pull Request Process

1. Fork the repository and create your feature branch from `main` or `master`.
2. Ensure you have [Git LFS](https://git-lfs.github.com/) installed locally to correctly fetch `.joblib` and `.pkl` model tracking files.
3. Install dependencies using `pip install -r requirements.txt`. (A virtual environment is recommended).
4. Verify your changes locally by running `streamlit run app.py`.
5. Issue a Pull Request detailing the purpose of your changes and any relevant execution instructions.

We look forward to reviewing your contributions!
